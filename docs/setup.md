# Setup Guide

## Prerequisites

- AWS account
- Node.js 22+
- [AWS CLI](https://docs.aws.amazon.com/cli/latest/userguide/install-cliv2.html) and [Session Manager plugin](https://docs.aws.amazon.com/systems-manager/latest/userguide/session-manager-working-with-install-plugin.html)
- Cloudflare account (optional, for custom domain)

## 1. Create IAM User for GitHub Actions

**AWS Console → IAM → Users → Create user**

- Username: `github-actions-sense-platform`
- Console access: **No**

Attach policies:
```
AWSLambda_FullAccess
AmazonAPIGatewayAdministrator
AWSCloudFormationFullAccess
IAMFullAccess
AmazonS3FullAccess
SecretsManagerReadWrite
AmazonEC2ContainerRegistryFullAccess
AmazonSSMFullAccess
AmazonVPCFullAccess
AmazonRDSFullAccess
```

**Security credentials → Create access key → Application running outside AWS**

Copy both keys immediately — you won't see the secret again.

## 2. Add GitHub Secrets

**Repo → Settings → Secrets and variables → Actions → New repository secret**

| Secret | Value |
|--------|-------|
| `AWS_ACCESS_KEY_ID` | From IAM user above |
| `AWS_SECRET_ACCESS_KEY` | From IAM user above |
| `AWS_ACCOUNT_ID` | Your 12-digit AWS account number |

Database credentials and API key are auto-generated by CDK and stored in AWS Secrets Manager — no manual secrets needed.

Find your account number: AWS Console → top right → account dropdown.

## 3. Deploy Infrastructure

**GitHub → Actions → Sense Platform CI/CD → Run workflow → main → Run workflow**

This provisions the full stack:
- VPC, subnets and security groups
- RDS PostgreSQL instance (isolated subnet, no public access)
- Database schema and seed data (via migration Lambda)
- Ingest Lambda (FastAPI/Mangum)
- API Gateway (rate-limited, CORS locked to frontend domain)
- S3 static frontend bucket
- Bastion host for database access (stopped by default)
- Secrets Manager entries (DB credentials + API key)

Takes ~15 minutes on first run (RDS provisioning takes time).

Stack outputs are in the **Show outputs** step when complete, or run:

```bash
aws cloudformation describe-stacks \
  --stack-name SensePlatformStack \
  --query 'Stacks[0].Outputs' --output table
```

## 4. Custom Domain (Optional)

1. Set `frontendDomain` in `cdk.json` to your domain
2. Redeploy infrastructure
3. CNAME your domain to the `FrontendUrl` from stack outputs (DNS only, no proxy)

Frontend configuration (API URL, site name) is auto-generated by CI at deploy time — no manual config needed.

## 5. Get Your API Key

The API key is needed for the sensor device to POST readings. Read endpoints are public.

```bash
aws secretsmanager get-secret-value \
  --secret-id sense-platform/api-key \
  --query SecretString --output text
```

## 6. Test with Simulated Data

Before your sensor device is ready, use the simulator to generate realistic readings. The script auto-discovers the API URL and key from CloudFormation:

```bash
./scripts/faker.sh dry-run           # Print payload without sending
./scripts/faker.sh once              # Send a single reading
./scripts/faker.sh start             # Start sending readings in background (every 60s)
./scripts/faker.sh start 30          # Custom interval (every 30s)
./scripts/faker.sh status            # Check if running + recent logs
./scripts/faker.sh stop              # Stop the background loop
```

## 7. Database Access

Use the bastion helper script for port forwarding to the RDS instance:

```bash
./scripts/bastion.sh start    # Start bastion + port forward (localhost:5432)
./scripts/bastion.sh creds    # Print DB credentials
./scripts/bastion.sh stop     # Stop the bastion when done
```

Connect your database client (e.g. TablePlus) to `localhost:5432` using the credentials from `creds`.

## Done

Every push to `main` deploys backend code and frontend automatically.
Infrastructure changes require manually triggering the workflow.
